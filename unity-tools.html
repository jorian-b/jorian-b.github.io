Open source
Utilities


During my time at Rhinox, I collaborated on utility libraries with a colleague, which have grown significantly in scope and functionality. Crafted to streamline project development, these tools have become integral to our workflow, enhancing efficiency and ensuring consistency. They tackle common challenges, from complex algorithms to data management, driving innovation and accelerating project delivery.

I also contributed to several other closed-source packages during my time at Rhinox. However, due to their proprietary nature, I am unable to provide further details about these projects.

Lightspeed

A lightweight coding extensions library for unity/C# offering lots of extensions, utility methods & other helpers.

Gui Utils

Same as Lightspeed but focused on the editor. Contains lots of utilities, common base classes, and even a completely custom drawing system for unity's editor.

Utilities

Lightspeed and GuiUtils are just utilities and do not change the behaviour of unity as is. Utility uses both to provide a bunch of additions like custom windows, custom inspectors, and a bunch of common monobehaviours.

Magnus

A general purpose framework that adds various features like services, players, level loading, loading scene, etc, to Unity.

Magnus Tasks

An extension for the magnus framework that provides you with a very flexible TaskManager, which allows you to setup sequential steps to complete.

Vortex

A generic implementation for a datalayer in Unity. The open source version only supports simple file saving, but it is easily extended to handle whatever you like.

Rendering

A few basic utilities for differentiating between Render Pipelines & some common sub shaders / URP base classes.

Perceptor

A logging framework that allows you to flexibly add channels and then choose which log level you wish to see per channel.
